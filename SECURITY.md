# Security Policy

## Supported Versions

Security updates are provided only for versions marked with :white_check_mark: in the table below.

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | :white_check_mark: |
| 5.0.x   | :x:                |
| 4.0.x   | :white_check_mark: |
| < 4.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately by emailing **security@example.com**.
Do **not** open a public issue for suspected vulnerabilities.

Please include, where possible:
- A description of the issue and impacted component(s)
- Steps to reproduce or a proof of concept
- Potential impact and suggested remediation (if known)

What to expect after you report:
- **Acknowledgement:** within 3 business days
- **Initial triage update:** within 7 business days
- **Status updates:** at least every 14 days until resolution or closure

Our process is: acknowledge report, validate and triage severity, develop and test a fix, coordinate disclosure, and publish a security update when appropriate. If a report is declined, we will provide a brief rationale.
